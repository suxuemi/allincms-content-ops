# Codex Round v0.4.0-r1 — Material Ingestion + H1 Dedup + Internal Links

Date: 2026-06-26
Subject: review of v0.4.0 design (Phase 2.5 ingestion guidance + body H1 duplicate fix + internal links suggester)
Source: user reported real bugs after using v0.3.0 — AI-drafted articles had H1 duplicated into body, never set internal links, and lacked rich company/product context.

## Findings + responses

### Fclassification.1 [high] WARN_PREFIXES changes audit contract
- Status: **applied**
- Resolution: warnings emit with `! ` prefix (not `- `), separate summary line `files=N issues=M warnings=W mode=full`. Exit code remains 0 when no blocking issues — backward compatible with v0.3 CI that greps the existing `issues=N` line. CHANGELOG documents the new `warnings=N` line.

### Fclassification.2 [med] H1 check would flood existing drafts with warnings
- Status: **applied**
- Resolution: `body_h1_duplicate` check fires only when frontmatter has `created_with_version >= 0.4.0`. `new_draft.py` injects this field. v0.3 drafts have no field → check skipped — no warning flood.

### Fprocess.1 [high] Phase 2.5 collides with Phase 2 paths
- Status: **applied**
- Resolution: Phase 2.5 conditional trigger — only fires on Path B-2 (fetch failed / partial) or Path C (cold). Path A / A′ / successful B-1 skip Phase 2.5 entirely. Before asking, AI lists what's already in `raw/` so user adds to / doesn't duplicate.

### Fprocess.2 [med] ingest_sources.py silent extraction failure
- Status: **applied**
- Resolution: `ingest_sources.py` exits non-zero with explicit stderr (`install pdftotext / pandoc OR convert to .md first and re-run`) when extraction returns "unavailable" body. Phase 2.5 brief acknowledges this fallback.

### Fhistorical.1 [high] AI drafts may overwrite human-curated wiki/products/ and wiki/personas/
- Status: **applied**
- Resolution: AI bootstrap drafts land at `wiki/products/_ai-drafts/<slug>.md` and `wiki/personas/_ai-drafts/<slug>.md` (subdirectory). User promotes by manually moving + renaming. `library_health.py` and `audit_content.py` exclude `_ai-drafts/` from `consumed_by` / orphan / stale checks. `SKILL.md` documents the promote step.

### Ffalsifiability.1 [med] No persisted regression checks for new behaviors
- Status: **applied**
- Resolution: `WARN_PREFIXES` is a module-level `frozenset` constant in `audit_content.py`; added `audit_skill_meta.py` checks for both the frozenset existence and the `body_h1_duplicate` warning prefix. Added a fixture-based smoke test for `suggest_internal_links.py` under `tests/test_suggest_links.py`.
