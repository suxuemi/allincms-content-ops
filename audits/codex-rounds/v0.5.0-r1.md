# Codex Round v0.5.0-r1 — Newbie First-Mile Optimization

Date: 2026-06-26
Subject: review of v0.5.0 design (doctor.py / onboarding-mode / examples / README first-mile entry)
Source: user goal "从用户实际使用的角度，优化流程和细节。便于新手也能快速上手"

## Findings + responses

### Fclassification.1 [high] --onboarding flag breaks audit↔reviewer contract
- Status: **applied** (with redesign — Design B abandoned)
- Resolution: removed `--onboarding` flag from `audit_content.py`. Replaced with `scripts/check_draft.py <slug>` — a teaching tool that scans a draft and explains each missing field + suggested fill. **Does not relax any blocking rule**; reviewer protocol stays intact. Newbies learn by example, not by exemption.

### Fclassification.2 [med] N=1 onboarding-per-repo limits expansion to sub-sites
- Status: **applied (moot)**
- Resolution: with Design B dropped, this limitation no longer exists. New language / sub-site additions follow the normal flow with `check_draft.py` guidance.

### Fprocess.1 [high] doctor.py output tiers collide with tooling-matrix
- Status: **applied**
- Resolution: `doctor.py` output reuses `tooling-matrix.md` vocabulary — three tiers `strong` / `degraded` / `missing` (not 5 ad-hoc classes). Each finding links to a `tooling-matrix.md` anchor for remediation. Output cached to `.doctor-cache.json` (gitignored). `first-contact.md` Phase -1 reads cache if present; if absent or stale (> 24h), prompts user to re-run doctor.

### Fprocess.2 [med] auto-writing backlog from script bypasses lessons protocol
- Status: **applied (moot)**
- Resolution: with `--onboarding` dropped, no auto-backlog writes. `check_draft.py` only prints to stdout — user copies the suggestions, no scripted side-effects to lessons / backlog.

### Fhistorical.1 [high] examples/ identity fields can leak to real publish
- Status: **applied**
- Resolution: `examples/sample-article-zh/` and `examples/sample-article-en/` use `__REPLACE_ME__` literal for all identity fields (site_id, canonical_url, author, allincms.*). `audit_content.py` gains a **new Hard Gate**: frontmatter containing `__REPLACE_ME__` blocks publish. Since no existing draft uses this literal, the gate is safe to add (decision tree Q1: no previously-passing draft fails).

### Fhistorical.2 [med] README insertion breaks v0.2+ users' muscle memory
- Status: **applied**
- Resolution: README structure preserves the existing "贴 prompt" code block in its original position. The "第一次用？" doctor section goes in a **new `## 第一次用？` heading above the prompt section**, separated by `---`. Header explicitly says "已经跑过？跳到下一段". Old users skipping = fine; new users see the new path.

### Ffalsifiability.1 [high] N=1 judgment via index.md row count is fragile
- Status: **applied (moot)**
- Resolution: with `--onboarding` dropped, no row-count judgment needed. Future "首篇豁免"-style features must use `git log --diff-filter=A` per F.falsifiability suggestion, not file content.

## Verification

- `__REPLACE_ME__` Hard Gate is regression-checked by `audit_skill_meta.py`.
- `doctor.py` cache schema covered by `audit_skill_meta.py`.
- `check_draft.py` is teaching-mode only; never modifies files.
- `examples/` README explicitly warns against copying identity fields.

## Design lineage

v0.5 dropped Design B (--onboarding) in favor of Design C+++ (check_draft.py teaching + examples sentinel). Net effect: same newbie outcome (less stuck on first publish), without touching v0.4 reviewer protocol or audit blocking semantics.
