# Codex Round v0.6.0-r1 — UX texture refinement from real-run screenshot

Date: 2026-06-26
Subject: review of v0.6.0 design (preamble compression + progress hints + protocol non-narration + doctor translation)
Source: user posted real screenshot of v0.5.0 AI session showing 8-line dense preamble, no progress on Phase 2 questions, raw doctor tier output, and AI meta-narrating the protocol.

## Findings + responses

### Ffalsifiability.1 [high] "Don't narrate" is a runtime behavior rule, can't be greppedfor regression
- Status: **applied**
- Resolution: rule split into three pieces. (a) `first-contact.md` § **Self-check before sending first message** lists 5 explicit banned phrases. (b) `audits/protocol-leak-samples.md` created — every release pastes at least one real first-message screenshot annotated leak/clean. (c) `audit_skill_meta.py` adds `protocol-leak-samples-fresh` check: file's last entry must be ≤ 60 days old.

### Ffalsifiability.2 [med] Progress (X/N) is hardcoded but N is dynamic
- Status: **applied**
- Resolution: `first-contact.md` Phase 2 explicitly defines `N = 3 + (Current Site empty field count from doctor)`. Includes a worked example: "缺 Browser profile → N=4". AI computes N from doctor cache, not hardcoded.

### Fclassification.1 [med] Removing "已读 SKILL.md + first-contact.md" loses contract anchor
- Status: **applied**
- Resolution: anchor preserved as a single line: `✓ 装好了 (v0.5.0 @ 6a8fbaf · SKILL.md + first-contact.md)`. The verb "已读取" is dropped (it's noise); the version + short SHA + the names of the loaded contract files stay. Pin users can verify AI is running their pinned version.

### Fclassification.2 [low] "Don't narrate" overfits — could hide breaking changes
- Status: **applied**
- Resolution: § Don't narrate gets an explicit whitelist of **必须报告**: writing files / changing wiki / installing dependencies / running git push / modifying settings. **禁止报告**: which Phase, which reference being read, protocol next step. This prevents the rule from silencing legitimate side-effects.

### Fprocess.1 [high] doctor-translation.md vs tooling-matrix.md double-source
- Status: **applied**
- Resolution: do NOT create `doctor-translation.md`. Instead, add a `user_facing_phrasing` column to the tooling-matrix tier table. doctor.py reads this column to emit the user-facing line. Single source of truth.

### Fprocess.2 [med] Substantive Phase -2 change might break v0.5 audit_skill_meta checks
- Status: **applied**
- Resolution: v0.6 PR explicitly enumerates which existing audit_skill_meta checks were touched (none broken — patterns are loose enough). New checks added: `first-contact-self-check`, `first-contact-progress-formula`, `protocol-leak-samples-fresh`, `tooling-matrix-user-phrasing-column`. CHANGELOG documents.

### Fhistorical.1 [low] v0.5 users anchored on the 8-line preamble
- Status: **applied**
- Resolution: 3-line preamble ends with `（详细诊断: python3 allincms-content-ops/scripts/doctor.py -v）` — anyone wanting the long version has an out. CHANGELOG v0.6.0 opens with "首屏从 8 行压到 3 行是有意的" note.

## Rejected findings

(none — all 7 applied with the design adjustments above)

## Verification

- Banned phrase list in `first-contact.md` covers the 5 phrases that appeared in the real-run screenshot.
- N formula has a worked example; AI cannot legitimately invent N anymore.
- `protocol-leak-samples.md` seeded with the v0.5 screenshot as the first leak example, so future regressions have a baseline.
- `tooling-matrix.md` gains `user_facing_phrasing` column for every tier × category cell.
