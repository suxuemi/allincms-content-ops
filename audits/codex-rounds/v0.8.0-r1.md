# Codex Round v0.8.0-r1 — MarkItDown + PaddleOCR + R2 + host-aware browser

Date: 2026-06-26
Subject: review of v0.8.0 design (MarkItDown 主推 + PaddleOCR 兜底 + R2 图床 + discovery host-aware browser + 6 个潜在问题)
Source: user real-use feedback after v0.7 — v0.7 doctor 红字"pdftotext/pandoc 缺失"显得吓人；要 Cloudflare R2 作 PicGo 替代；discovery 没区分 Codex 内置浏览器 vs Claude Code Chrome MCP。

## Findings + responses

### Fclass.1 [high] PaddleOCR 1GB+ 依赖被默认 strong 判定，"兜底"伪装成"基线"
- Status: **applied**
- Resolution: doctor `chinese_ocr × ingest` 改为 `optional` 类别（**不计入 degraded 红字汇总**）。只在 ingest_sources.py 实际遇到图片/扫描 PDF 抽取失败时，在该次 ingest 报告里点名建议装。`ocr-strategy.md` 显式写 "按需启用，不进 doctor 红字"。

### Fclass.2 [high] MinerU AGPL-3.0 风险传染
- Status: **applied**
- Resolution: `references/ocr-strategy.md` 里 MinerU 行加 `license: AGPL-3.0 — 不推荐集成进闭源服务，仅可独立 CLI 使用`。`ingest_sources.py` 永不自动调 MinerU——只在 user 主动指定时手动运行。

### Fclass.3 [med] R2 credentials 可能被日志/manifest 泄露
- Status: **applied**
- Resolution: `r2_upload.py` 对 `access_key_id` 做 `xxx...xxx` 中段省略；`secret_access_key` 永不出现在任何 print/log；所有 boto3 client 构造走 logger 不走 print；setup wizard 完成后打印一行 "凭证已写入 ~/.config/...，不会出现在任何 stdout/manifest"。

### Fproc.1 [high] ingest fallback 链"<30字"判定太脆
- Status: **applied**
- Resolution: 阈值改 **双闸**：`输入文件 > 50 KB 且 输出 < 100 字` 才算"短到要兜底"；`EXTRACTION_MIN_CHARS = 100` 命名常量 + fixture 注释 + tests 锁。每次抽取写 `extraction_log: {tool, chars, path}` 进 manifest；原 markitdown 输出保留在 `.raw/<source>.markitdown.md`，paddleocr 兜底输出在 `.raw/<source>.paddleocr.md`，永不覆盖。

### Fproc.2 [high] discovery host detection 靠"猜"会卡死
- Status: **applied**
- Resolution: `current-site-discovery.md` Step 3 改 **explicit confirm**：AI 先打印 `我打算用 <browser>，依据是 <signal>。30 秒内回车确认，或输入 'manual' 让我给 URL 让你自己开`。第 5 行加 `Unknown host → manual: 列出 URL 让用户自己开 + 贴 HTML 进来 → AI 从粘贴的 HTML 走 Step 4-5`。

### Fhist.1 [med] PicGo / R2 混用 → media/index.md 多域名
- Resolution: `references/media-pipeline.md`（新文件）写"图床一旦选定，同项目不切换；切换需跑 `scripts/migrate_media.py`（留到 v0.9+）"；doctor 加 `media_mix × media` 检查——media/index.md 出现 ≥2 个不同 host 域名时 warn "图床混用，新老文章引用可能漂移"。

### Ffalsif.1 [med] doctor ANY-of + 阈值都没 regression
- Status: **applied**
- Resolution: 新增 `tests/test_doctor_extraction_matrix.py` 覆盖 4 组合（都无 / 只 markitdown / 只 CLI / 双有）的 doctor 输出；阈值改命名常量 `EXTRACTION_MIN_CHARS = 100` + 注释指向 `tests/fixtures/short_pdfs/` 的 ground truth 中位长度。

## Rejected findings

(none — all 7 applied; PaddleOCR & MinerU concerns 都按 codex 建议降权)

- doctor `chinese_ocr` 在不调用 ingest 的常规 session 永不被报 degraded
- MinerU 永不被 ingest_sources 自动触发
- R2 access_key_id 中段省略；secret_access_key 不进 log
- ingest 双闸 50KB×100chars 写进 EXTRACTION_MIN_CHARS 常量 + fixture
- discovery Step 3 改 explicit confirm + Unknown host manual fallback
- media/index.md 检测多 host → doctor warn
- 4 组合 doctor 状态由 unit test 锁住
